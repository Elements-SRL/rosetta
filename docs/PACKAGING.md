# Building the Windows installer

`rosetta` links dynamically against 4 vendor DLLs (`e384c.dll`, `FTD2XX.dll`,
`MPSSE.dll`, `okFrontPanel.dll`) and needs the Microsoft VC++ runtime at run time.
Committed copies of these live under `vendor\dlls\` and `vendor\redist\` so the
installer build is reproducible from a clean clone, with no dependency on the
sibling `e384c` repo or any developer-machine paths.

The release process produces a single `RosettaSetup.exe` that:
1. Silently installs the VC++ Redistributable (x64) if it isn't already present.
2. Installs `rosetta.exe` plus all 4 DLLs into the same folder under
   `Program Files\rosetta\bin\`.

## One-time setup

- Install the WiX v3.14 toolset (`candle.exe`/`light.exe`). Either install it
  system-wide, or download the binaries zip from
  https://github.com/wixtoolset/wix3/releases/tag/wix3141rtm and unzip it
  somewhere (e.g. `C:\wix-toolset\bin`).
- `cargo install cargo-wix`

## Updating the vendored DLLs / VC++ redistributable

- `vendor\dlls\*.dll` — replace with fresh copies built from the `e384c` project's
  **Release** config (and, for the FTDI/Opal Kelly DLLs, from their SDK/driver
  install) whenever the vendor SDK is updated.
- `vendor\redist\vc_redist.x64.exe` — Microsoft's official, redistributable
  installer. Refresh periodically from https://aka.ms/vs/17/release/vc_redist.x64.exe.

## Release build steps

Run from the repo root (adjust `<WIX_BIN>` to wherever `candle.exe`/`light.exe` live,
and `<VERSION>` to match the version in `Cargo.toml`):

```
cargo build --release
cargo wix -b <WIX_BIN>
<WIX_BIN>\candle.exe installer\bundle.wxs -ext WixBalExtension -ext WixUtilExtension -dVersion=<VERSION> -out target\wix\bundle\
<WIX_BIN>\light.exe target\wix\bundle\bundle.wixobj -ext WixBalExtension -ext WixUtilExtension -out target\wix\RosettaSetup.exe
```

The final distributable is `target\wix\RosettaSetup.exe` — hand this single file to
end users.

Note: `bundle.wxs` deliberately lives in `installer\`, not `wix\` — `cargo wix`
auto-compiles every `.wxs` file it finds directly in `wix\`, and having both the
`<Product>` (MSI) and `<Bundle>` definitions in that folder makes `cargo wix` try to
link them together and fail with `LGHT0089: Multiple entry sections`. Keeping
`bundle.wxs` in a separate folder avoids that; its relative paths (e.g.
`vendor\dlls\...`, `target\wix\rosetta-<VERSION>-x86_64.msi`) are resolved against the
current working directory when you invoke `candle`/`light`, not against the `.wxs`
file's own location, so running the commands above from the repo root just works.

## Files

- `wix\main.wxs` — MSI definition (generated by `cargo wix init`, then hand-edited to
  add the 4 vendor DLLs alongside `rosetta.exe`).
- `installer\bundle.wxs` — WiX Bundle (Burn) chaining the VC++ redistributable and the
  MSI into one `setup.exe`.
